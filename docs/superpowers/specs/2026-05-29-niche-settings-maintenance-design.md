# Niche maintenance from Settings — Design Spec

**Date:** 2026-05-29  
**Status:** Implemented — plan at `docs/superpowers/plans/2026-05-29-niche-settings-maintenance.md`  
**Scope:** Move all niche maintenance (catalog refresh + market scan) to Settings; remove weekly scheduler and Niches page scan trigger. First bootstrap/scan run is sufficient for day-to-day triage — operators trigger updates manually when needed.

**Related:** [Niche opportunity scanner](2026-05-27-niche-opportunity-scanner-design.md), [`niches:bootstrap`](2026-05-27-niches-bootstrap-design.md), [Niches operational reference](../../niches.md), [Laravel Cloud deployment](../../deployment/laravel-cloud.md).

**Approach:** Queue existing Artisan commands from Settings (`Artisan::queue`). Add `--force` to `niches:bootstrap` for non-interactive overwrite. No DB catalog migration.

---

## Goal

Operators should control when niche data is refreshed:

1. **Catalog refresh** (`niches:bootstrap`) — re-fetch UK cities and Google Places taxonomy, validate in Birmingham, overwrite `config/niches.php`.
2. **Market scan** (`niches:scan`) — dispatch GBP sample jobs for all configured niche×city pairs.

Both actions live on **Settings** only. No weekly scheduler. No **Run Now** on the Niches index.

CLI commands remain for power users and local development.

---

## Decisions (brainstorming)

| Topic | Decision |
|-------|----------|
| Scope | Both bootstrap and scan from Settings (option C) |
| Niches page | Remove **Run Now**; keep **Manage niches** (ignore/include) |
| Scheduler | Remove `Schedule::command('niches:scan')` from `routes/console.php` |
| Implementation | Queue existing Artisan commands — no new service layer |
| Catalog storage | Stay in `config/niches.php` (not DB) |
| Scan UI options | **Force re-scan** checkbox only (`--force`); no city/niche pickers in v1 |
| Bootstrap UI options | Typed confirmation `REFRESH`; fixed `--min-results=5`; no dry-run in v1 |
| Bootstrap non-interactive | New `--force` flag skips overwrite `confirm()` prompt |
| Cloud caveat | Settings warns: ephemeral filesystem — commit and redeploy config after bootstrap |
| Progress feedback | Flash "queued" only; failures in Horizon/logs |

## Architecture

```text
Settings page
  ├── GET  /settings
  │     └── nicheMaintenance stats (counts, last scan, config date)
  ├── POST /settings/niches/scan
  │     └── Artisan::queue('niches:scan', [--force]?)
  └── POST /settings/niches/bootstrap
        └── Artisan::queue('niches:bootstrap', --no-interaction --force)

Removed:
  ├── Schedule::command('niches:scan')  (routes/console.php)
  ├── POST /niches/scan                 (routes/web.php)
  └── NicheScanController::trigger
```

## Settings UI

New card **Niche maintenance** below **Defaults**.

### Read-only stats

| Stat | Source |
|------|--------|
| Niches configured | `count(config('niches.niches', []))` |
| Cities configured | `count(config('niches.cities', []))` |
| Last market scan | `NicheScan::max('ran_at')` — `diffForHumans()` or "Never" |
| Config generated | Parse `// Generated by niches:bootstrap on YYYY-MM-DD` from `config/niches.php` header, or "Unknown" |

### Run market scan

- Button: **Run market scan** (secondary, not primary)
- Checkbox: **Force re-scan** → `--force` when checked
- Helper: "Dispatches sample scans for all configured niche×city pairs (respects ignored niches). A full catalog is ~6,000 queue jobs."
- Submit → `POST /settings/niches/scan` with `{ force: boolean }`
- Success flash: "Market scan queued."

### Refresh catalog

- Button: **Refresh catalog** (destructive styling)
- Confirmation: operator must type `REFRESH` in a text field to enable submit
- Warning: "Re-fetches UK cities and Google Places types, validates in Birmingham, and overwrites `config/niches.php`. On Laravel Cloud, commit and redeploy the updated config for changes to persist."
- Submit → `POST /settings/niches/bootstrap` with `{ confirm: 'REFRESH' }`
- Success flash: "Catalog refresh queued."

## Backend

### `SettingsController::index`

Add `nicheMaintenance` to Inertia props:

```php
'nicheMaintenance' => [
    'niche_count'      => count(config('niches.niches', [])),
    'city_count'       => count(config('niches.cities', [])),
    'last_scan_at'     => NicheScan::max('ran_at')?->toISOString(),
    'last_scan_human'  => $lastScan ? $lastScan->diffForHumans() : 'Never',
    'config_generated' => self::parseNichesConfigDate(), // regex on config file header
],
```

### `SettingsController::scanNiches`

```php
$validated = $request->validate(['force' => 'sometimes|boolean']);
$params = ! empty($validated['force']) ? ['--force' => true] : [];
Artisan::queue('niches:scan', $params);
return back()->with('success', 'Market scan queued.');
```

### `SettingsController::bootstrapNiches`

```php
$request->validate(['confirm' => 'required|in:REFRESH']);
Artisan::queue('niches:bootstrap', [
    '--no-interaction' => true,
    '--force'          => true,
]);
return back()->with('success', 'Catalog refresh queued.');
```

### `NichesBootstrapCommand`

Add to signature: `{--force : Overwrite config/niches.php without confirmation}`

In `writeConfig()`, skip the overwrite prompt when `--force` or `--no-interaction` is set:

```php
if (file_exists($path) && ! $this->option('force') && ! $this->option('no-interaction')) {
    if (! $this->confirm('config/niches.php already exists. Overwrite?', false)) {
        $this->warn('Config write skipped (declined overwrite).');
        return 'skipped (declined overwrite)';
    }
}
```

Settings always queues with `--no-interaction --force`, so the web path always overwrites (unless `--dry-run`).

## Error handling

| Case | Behaviour |
|------|-----------|
| `confirm` not `REFRESH` | 422 validation error on Settings form |
| Unauthenticated | Redirect to login (existing `auth` middleware) |
| Queued command failure | Logged / visible in Horizon; no Settings inline error |
| Bootstrap on ephemeral host | Config written at runtime but lost on redeploy — documented in UI |
| Missing `GOOGLE_PLACES_API_KEY` during bootstrap | Existing command behaviour: skip validation, write unvalidated list |

## Out of scope (v1)

- DB-backed niche catalog (ephemeral-cloud persistence via database)
- Dry-run bootstrap from UI
- City/niche subset pickers on Settings
- In-app progress polling for queued maintenance commands
- Scheduled `niches:bootstrap`
- Removing CLI commands

## Operator workflow (after implementation)

1. **Day-to-day:** Use Niches index to triage markets; ignore low-quality niches via Manage panel.
2. **Refresh market data:** Settings → Run market scan (optionally force).
3. **Expand catalog:** Settings → Refresh catalog → commit updated `config/niches.php` → deploy (required on Laravel Cloud).
